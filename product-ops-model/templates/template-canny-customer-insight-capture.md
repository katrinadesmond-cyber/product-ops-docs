# Canny Customer Insight Capture Template

## Purpose

Capture customer signals in a consistent structure so they can be synthesized into validated insights and opportunities.

## 1) Post title (standard format)

`[Theme] - [Customer/Segment] - [Problem signal]`

Example: `Ad Serving - Woolworths - Budget pacing unpredictability`

## 2) Signal description (required)

- **What was observed/heard?**  
  (fact-based statement only)
- **Context**  
  (where/when this happened)
- **Who said it / source**  
  (customer name, role, call/workshop/ticket)
- **Evidence artifact**  
  (recording link, notes, ticket, dashboard snapshot)

## 4) Signal quality checks

- [ ] Signal is specific and not generic.
- [ ] Source is identifiable and verifiable.
- [ ] No solution proposal is mixed into signal.
- [ ] Wording is neutral (no interpretation inflation).

## 6) Insight promotion criteria

Promote from signal -> insight when:
- repeated across multiple sources;
- impact is material (customer and/or business);
- confidence is supported by evidence.
