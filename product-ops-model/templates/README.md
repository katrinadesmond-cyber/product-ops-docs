# Templates Index (Confluence-Friendly Markdown)

Use these templates as standardized artifact starting points.

## Available templates

1. `template-opportunity-assessment.md`
2. `template-discovery-brief.md`
3. `template-experiment-card.md`
4. `template-prd-lite.md`
5. `template-quarterly-product-health-review.md`

## Suggested mapping to Confluence

- Create a Confluence space section named `Product Ops Templates`.
- Create one page per template and copy Markdown content.
- Add labels per template type (for example: `opportunity`, `discovery`, `experiment`, `prd`, `health-review`).

## Working agreement

- Keep template headings intact for consistency across teams.
- Add context in free-text sections; do not remove required fields.
- Link related artifacts (opportunity -> discovery -> experiment -> PRD -> learning review).
