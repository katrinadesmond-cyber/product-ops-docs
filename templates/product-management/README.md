# Product Management Templates

This folder contains Markdown conversions of your Confluence product operations templates.

## Files

- `pitch-template-target-opportunity.md`  
  Use for opportunity pitch reviews with problem framing, scope appetite, assumptions, and success measures.

- `pitch-template-one-pager.md`  
  Lean one-pager for delivery-focused cycles and cross-team execution alignment.

- `product-opportunities-assessment.md`  
  Opportunity evaluation framework (go/no-go thinking and discovery principles).

- `customer-insight-framing.md`  
  Structured method for capturing customer insight signals and turning them into opportunities.

- `product-go-to-market-strategy-template.md`  
  GTM strategy template including positioning, messaging, channels, and value matrix.

- `product-gtm-launch-plan-template.md`  
  Cross-functional launch planning template covering readiness, execution, dependencies, and risk.

- `product-gtm-release-plan-template.md`  
  Technical/product release planning template with release type, rollout planning, UAT, and hypercare.

- `product-messaging-framework-template.md`  
  Messaging framework for audience, value proposition, proof points, and tone.

- `release-notes-template.md`  
  Customer-facing release notes template for feature summaries and user impact.

## Best Practices for AI-Assisted Workflows

- Keep section headings unchanged to improve prompt and context consistency.
- Keep placeholders explicit (for example, `[target segment]`) until finalized.
- Add links to supporting artifacts (research, dashboards, specs) directly in each section.
- Track decisions in Git commits so rationale is preserved over time.
