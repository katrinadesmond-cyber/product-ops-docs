# AI Product Operations Framework

This repository contains a practical **Product Operations model** for product teams that are integrating AI into the product development cycle.

It is designed to help teams:
- scale decision making and delivery with shared standards;
- keep strong continuous discovery habits;
- use AI responsibly across planning, discovery, build, and learning;
- adopt reusable templates (Markdown, Confluence-friendly);
- establish clear Cursor skills and rules.

## Repository structure

- `product-ops-model/01-ai-product-operations-model.md`  
  Core operating model, roles, service model, and governance.
- `product-ops-model/02-foundations-and-standards.md`  
  Non-negotiable standards, quality gates, and artifact requirements.
- `product-ops-model/03-continuous-discovery-operating-system.md`  
  Continuous discovery operating system and routines.
- `product-ops-model/04-implementation-roadmap.md`  
  30-60-90 day rollout and maturity path.
- `product-ops-model/05-shape-up-plc-alignment.md`  
  Alignment between current Shape Up PLC (Confluence) and this framework.
- `product-ops-model/06-skills-knowledge-training-guide.md`  
  Role-based training plan for scalable execution.
- `product-ops-model/07-team-learnings-and-priorities-mar-2026.md`  
  Interview-derived learnings, risks, and 90-day priority actions.
- `product-ops-model/08-cursor-rules-and-skills-governance.md`  
  Clear definitions, ownership, and lifecycle for Cursor rules and skills.
- `product-ops-model/templates/`  
  Ready-to-use Markdown templates for common product artifacts.
- `product-ops-model/cursor/skills-catalog.md`  
  Suggested AI skills for product teams using Cursor.
- `.cursor/rules/product-ops-rules.mdc`  
  Team rule set for consistent AI-assisted product work.

## How to use this framework

1. Start with `01-ai-product-operations-model.md` to align on intent and structure.
2. Agree the standards in `02-foundations-and-standards.md`.
3. Implement discovery practices in `03-continuous-discovery-operating-system.md`.
4. Select and socialize templates from `templates/`.
5. Set up Cursor rules from `.cursor/rules/product-ops-rules.mdc`.
6. Align current Shape Up practices via `05-shape-up-plc-alignment.md`.
7. Run enablement using `06-skills-knowledge-training-guide.md`.
8. Roll out with the plan in `04-implementation-roadmap.md`.

## Confluence migration note

All templates are plain Markdown with headings and checklists so they can be:
- copied directly into Confluence pages; or
- used in Git-based documentation workflows.
